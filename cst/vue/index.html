# 简介

# 描述

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是， Vue 被设计为可以自底向上逐层应用。 Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时， Vue 也完全能够为复杂的单页应用提供驱动。

如果你想在深入学习 Vue 之前对它有更多了解，我们制作了一个视频，带您了解其核心概念和一个示例工程。

如果你已经是有经验的前端开发者，想知道 Vue 与其它库/框架有哪些区别，请查看对比其它框架。

# 文件目录

build         webpack相关的配置
config        项目中的webpack的开发模式和生产模式的设置
node_modules  存放js包依赖
dist          存放生产环境打包后产生的文件
src           你的程序存放的地方
 |---assets     存放项目静态文件
 |---components     存放组件
 |---router     存放vue-router
 |---store     存放vuex
 |---App.vue    跟页面，可以在main.js中改掉
 |---main.js    程序入口，在这可以配置Vue各种依赖
 static       项目外静态文件
 test         测试报告相关
 .babelrc     babel相关设置
 .editorconfig  编辑器代码规范配置
 .eslintignore  eslint语法检测相关的设置，设置不对哪些文件进行语法校验
 .eslintrc.js   eslint语法检测的设置
 .gitignore     git相关的配置，设置不提交哪些git
 .postcssrc.js  webpack的postcss插件对css处理的配置
 index.html     模板页面，vue-cli会向这里的id="app"标签注入vue程序
 package.json   js包依赖管理文件
 package-lock.json  js包管理文件,与package.json不同的是,它有明确的各依赖版本号，实际安装的结构，也有逻辑树的结构
 README.md    简介文件

# 生命周期

# 描述

在Vue的生命周期有几种钩子函数，开发人员通过钩子函数的方法实现在vue的不同生命周期阶段执行对应的命令。这里引用一个著名的图：

# 生命周期函数

beforeCreate()  //在数据加载前触发。
created()       //在数据加载完成时候触发，此时data和method等已经可以访问到了。
beforeMount()   //在dom挂载之前触发，此时还不能访问到dom。
mounted()       //此时已经编译好模板，并且挂载到dom上。
beforeUpdate()  //在视图更新前触发，此时还没有开始更新dom，但是数据已经改变了。
updated()       //在视图已经更新完毕的时候触发，此时dom已经完成了更新。
beforeDestroy() //在当前组件销毁前触发，此时组件内的内容还是可以访问的。
destroyed()     //在当前组件销毁完成时候触发，在调用之后，vue会把当前组件所有的东西解绑、移除事件等、销毁实例等。
