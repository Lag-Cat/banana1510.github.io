# Promise

# 简介

在有些业务场景下需要嵌套多重的回调函数，导致代码可读性降低，给维护人员带来很大麻烦，这称为“回调地狱”。es6中提供了 Promise 对象，在一定程度上优化了这个问题。
Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。它由社区最早提出和实现，ES6 将其写进了语言标准，统一了用法，原生提供了 Promise 对象。

Promise有以下特点:

(1) Promise 对象代表一个异步操作， Promise 中有三个状态，分别是进行中( pending )、完成状态( fulfilled )和拒绝状态( rejected )。 (2) 一旦状态改变了，就不会再变，其状态只能从 pending 变为 fulfilled , 或者从 pending 变为 rejected 。 (3) Promise 一旦执行，就无法取消，无法在执行过程中中断。

# 基本用法

Promise 对象是一个构造函数，用来生成 Promise 实例。

let temp = false;
let promise = new Promise((resolve, reject) => {
    if (temp) {
        resolve('success');
    } else {
        reject('fail');
    }
})

Promise 可以接受一个函数作为参数，该函数中有 resolve 和 reject 两个参数。

其中 resolve 是将该 Promise 对象的状态从 pending 转为 fulfilled 状态，后面括号中可以带上成功结果的参数，执行 resolve 时会把参数传递出去。 reject 是将该 Promise 对象的状态从 pending 转为 reject 状态，后面括号中可以带上失败结果的参数，执行 reject 时会把参数传递出去。

在生成实例后，可以通过 then 来获取处理后的结果。如下：

//接上
promise.then(result => {
    console.log(result); //success
}, error => {
    console.log(error); //fail
})

# Promise.prototype.then

then 中可以传入两个函数作为参数，分别为成功时需要执行的操作和拒绝（错误）时执行的操作。 在这两个函数中，可以通过传入的参数来获取它们各自处理后返回的结果。

以下为简单的案例：

function promise() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve('success')
        }, 1000 * 2);
    })
}

promise().then(res => {
    console.log(res);
}, err => {
    console.log(err);
})
//'success'

function promise() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log("success in promise")
            resolve('success')
        }, 1000 * 2);

        setTimeout(() => {
            console.log("fail in promise")
            resolve('fail')
        }, 1000 * 4);
    })
}

promise().then(res => {
    console.log(res); //'success'
}, err => {
    console.log(err);
})

// success in promise
// success
// fail in promise

上面例子中，通过执行函数返回一个Promise实例，在这时用 then 可以获取到其返回的结果，在例子2中可以看出，状态只会改变一次。

Promise 新建时会立即执行

let promise = new Promise(function(resolve, reject) {
    console.log('Promise');
    resolve();
});

promise.then(function() {
    console.log('resolved.');
});

console.log('Hi!');

// Promise
// Hi!
// resolved

上面例子可以看出 Promise 新建时候就是马上执行，其结果会在同步执行代码后才执行 then 回调数据时获取到。

下面为异步调用例子（xhr请求)

function getData(url) {
    new Promise((resolve, reject) => {
        let xhr = new XMLHttpRequest();
        xhr.open('get', 'url', true)
        xhr.setRequestHeader("Content-Type", "application/json");
        xhr.withCredentials = true;
        xhr.onreadystatechange = function(e) {
            if (xhr.readyState == 4 && xhr.status == 200)
                resolve(xhr.responseText);
            else if (xhr.status == 500)
                reject('error');
        }
        xhr.send();
    })
}

getData('www.xxxx.com/getData').then(res => {
    console.log(res)
}, err => {
    console.log(err);
})

上面例子在xhr状态为4和返回结果状态为200的时候会将 Promise 状态转为 resolve 状态，并且返回请求的结果，如果返回结果状态为500的时候，会转为 reject 状态，并且返回错误信息。

# Promise.prototype.catch

Promise是.then(null, rejection)或.then(undefined, rejection)的别名，用于指定发生错误时的回调函数。

axios.get('www.xxxx.com/getdata').then(res => {

}).catch(err => {
    console.log(err);
})

上面例子中通过axios发送get请求，可以在then中处理请求成功的时候返回的数据，在catch中处理请求失败的信息。

let a = new Promise((resolve, reject) => {
    throw new Error('err');
})

a.then(res => {}).catch(err => {
    console.log(err)
})
//Error: err

catch可以捕获到promise中抛出的错误信息。 catch也可以捕获到reject中抛出的信息。

let a = new Promise((resolve, reject) => {
    reject('err2020')
})

a.then(res => {}).catch(err => {
    console.log(err)
})
//err2020

在链式调用中catch能捕获到执行期间出现抛出错误或者拒绝状态的结果。

function funA() {
    return new Promise((resolve, reject) => {
        resolve('2020-A')
    })
}

function funB() {
    return new Promise((resolve, reject) => {
        reject('2020-B')
    })
}

function funC() {
    return new Promise((resolve, reject) => {
        reject('2020-C')
    })
}

funA().then(res => {
    console.log(res, 'run success');
    return funB();
}).then(res => {
    console.log(res, 'run success');
    return funC();
}).catch(err => {
    console.log(err, 'run error');
})
//2020-A run success
//2020-B run error

一般来说，不要在then()方法里面定义Reject状态的回调函数（即then的第二个参数），总是使用catch方法

// bad
promise
    .then(function(data) {
        // success
    }, function(err) {
        // error
    });

// good
promise
    .then(function(data) { //cb
        // success
    })
    .catch(function(err) {
        // error
    });

上面代码中，第二种写法要好于第一种写法，理由是第二种写法可以捕获前面then方法执行中的错误，也更接近同步的写法（try/catch）。因此，建议总是使用catch()方法，而不使用then()方法的第二个参数。

跟传统的try/catch代码块不同的是，如果没有使用catch()方法指定错误处理的回调函数，Promise 对象抛出的错误不会传递到外层代码，即不会有任何反应。

const someAsyncThing = function() {
    return new Promise(function(resolve, reject) {
        // 下面一行会报错，因为x没有声明
        resolve(x + 2);
    });
};

someAsyncThing().then(function() {
    console.log('everything is great');
});

setTimeout(() => {
    console.log(123)
}, 2000);
// Uncaught (in promise) ReferenceError: x is not defined
// 123

但是其中也有个问题，就是 catch 无法捕获到 Promise 中异步处理时抛出的错误，如下：

let a = new Promise((resolve, reject) => {
    setTimeout(() => {
        throw new Error('err');
    }, 1000)
})

a.then(res => {}).catch(err => {
    console.log(err)
})
//Uncaught Error: err

因为 js 事件机制， setTimeout 在定义的时候会放到事件队列中，在 Promise 执行完后才会执行，而到了 setTimeout 执行的时候， Promise 已经执行完成，此时错误就会在外层抛出，致使 Promise 捕获不到 setTimeout 中的错误。js中的try{}catch{}处理亦是如此。

# Promise.prototype.finally

finally是 Promise 对象中不管是 resolve 还是 reject 状态，都会在 Promise 执行完后执行的函数。

let a = new Promise((resolve, reject) => {
    reject(err);
})

a.then(res => {
    //...
}).catch(err => {
    //...
}).finally(() => {
    //...
})

学过后端的同学对此肯定会很熟悉，跟后端try..catch..finally是一样使用的。

# Promise.all

Promise.all是用于吧多个Promise实例，包装成一个新的Promise的实例。 其用法是将多个Promise实例放到数组中，用Promise.all执行。当所有Promise实例完成之后会将该新的实例的状态转为 resolve ，当出现一个实例出错时，将马上执行 then 中的 reject 或 catch 。

function promise1() {
    return new Promise((resolve, reject) => {
        resolve('promise2020-1');
    })
}

function promise2() {
    return new Promise((resolve, reject) => {
        resolve('promise2020-2');
    })
}

function promise3() {
    return new Promise((resolve, reject) => {
        resolve('promise2020-3');
    })
}

Promise.all([promise1(), promise2(), promise3()])
    .then(res => {
        console.log(res, 'success')
    })
    .catch(err => {
        console.log(err, 'error')
    })
//[ 'promise2020-1', 'promise2020-2', 'promise2020-3' ] success

function promise1() {
    return new Promise((resolve, reject) => {
        resolve('promise2020-1');
    })
}

function promise2() {
    return new Promise((resolve, reject) => {
        reject('promise2020-2');
    })
}

function promise3() {
    return new Promise((resolve, reject) => {
        reject('promise2020-3');
    })
}

Promise.all([promise1(), promise2(), promise3()])
    .then(res => {
        console.log(res, 'success')
    })
    .catch(err => {
        console.log(err, 'error')
    })
//promise2020-2 error

上述例子中可以看出，当全部 Promise 成功执行的时候，新的 Promise 实例也会变为 resolve 状态。当其中一个 Promise 是 reject 状态的话，新 Promise 将会变为 reject 状态，并且会返回触发状态变换的那一个 Promise 返回的信息。

Promise.all中执行的 Promise 对象不会因为其中一个 Promise 变为 reject 状态而停止执行。

function promise1() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-1')
            resolve('promise2020-1')
        }, 1000 * 1);
    })
}

function promise2() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-2')
            reject('promise2020-2');
        }, 1000 * 2);
    })
}

function promise3() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-3')
            resolve('promise2020-3');
        }, 1000 * 3);
    })
}

Promise.all([promise1(), promise2(), promise3()])
    .then(res => {
        console.log(res, 'success')
    })
    .catch(err => {
        console.log(err, 'error')
    })

// promise2020-1
// promise2020-2
// promise2020-2 error
// promise2020-3

# Promise.race

Promise.race用于将多个 Promise 对象重新合并成一个新的 Promise 对象，与 Promise.all 不同的是， Promise.race 相当于一场竞赛，第一个改变状态的 Promise 对象将会影响到该新的 Promise 对象的状态。

function promise1() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-1')
            reject('promise2020-1')
        }, 1000 * 1);
    })
}

function promise2() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-2')
            resolve('promise2020-2');
        }, 1000 * 2);
    })
}

function promise3() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-3')
            resolve('promise2020-3');
        }, 1000 * 3);
    })
}

Promise.race([promise1(), promise2(), promise3()])
    .then(res => {
        console.log(res, 'success')
    })
    .catch(err => {
        console.log(err, 'error')
    })

// promise2020-1
// promise2020-1 error
// promise2020-2
// promise2020-3

# Promise链式调用

Promise链式调用，原理是每一个Promise对象执行完后在 then 或者 catch 中返回一个 Promise 对象。

function promise1() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-1')
            resolve('promise2020-1')
        }, 1000 * 1);
    })
}

function promise2() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-2')
            resolve('promise2020-2');
        }, 1000 * 1);
    })
}

function promise3() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log('promise2020-3')
            resolve('promise2020-3');
        }, 1000 * 1);
    })
}

promise1().then(res => {
    console.log(res);
    return promise2();
}).then(res => {
    console.log(res);
    return promise3();
}).then(res => {
    console.log(res);
}).catch(err => {
    console.log(err);
})

在catch中返回Promise对象实现链式调用

function promise1() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      console.log('promise2020-1')
      reject('promise2020-1')
    }, 1000 * 1);
  })
}
function promise2() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      console.log('promise2020-2')
      resolve('promise2020-2');
    }, 1000 * 1);
  })
}
function promise3() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      console.log('promise2020-3')
      resolve('promise2020-3');
    }, 1000 * 1);
  })
}

promise1().then(res => {
  console.log(res);
  return promise2();
}).catch(err => {
  console.log(err, 'catch1')
  return promise2();
}).then(res => {
  console.log(res)
  return promise3()
}).then(res => {
  console.log(res);
}).catch(err => {
  console.log(err, 'catch2')
})
